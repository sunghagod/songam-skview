# Google Sheets 연동 가이드 — 송암공원 SK VIEW

## 1. Google 스프레드시트 생성

1. [https://sheets.google.com](https://sheets.google.com) 접속 (로그인)
2. **빈 스프레드시트** 생성
3. 시트 이름: `관심고객` (선택)
4. **첫 번째 행(헤더)** 에 아래와 같이 입력:

   | A | B | C | D | E |
   |---|---|---|---|---|
   | 타임스탬프 | 이름 | 연락처 | 관심평형 | 문의내용 |

---

## 2. Google Apps Script 작성

1. 스프레드시트 상단 메뉴 → **확장 프로그램 → Apps Script** 클릭
2. 기존 코드를 **전부 삭제**하고 아래 코드를 붙여넣기:

```javascript
function doPost(e) {
  try {
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    var data  = JSON.parse(e.postData.contents);

    sheet.appendRow([
      new Date(),          // A: 타임스탬프
      data.name  || '',    // B: 이름
      data.phone || '',    // C: 연락처
      data.size  || '',    // D: 관심평형
      data.message || ''   // E: 문의내용
    ]);

    return ContentService
      .createTextOutput(JSON.stringify({ result: 'success' }))
      .setMimeType(ContentService.MimeType.JSON);

  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ result: 'error', message: err.toString() }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}
```

3. **Ctrl+S** 저장 (프로젝트 이름: `송암공원 관심고객`)

---

## 3. 웹 앱으로 배포

1. 오른쪽 상단 **배포** 버튼 클릭 → **새 배포**
2. 유형 선택 (톱니바퀴 아이콘) → **웹 앱** 선택
3. 설정:
   - 설명: `송암공원 관심고객 폼`
   - **실행 계정**: 나 (나의 Google 계정)
   - **액세스 권한**: **모든 사용자** ← 반드시 이것으로 설정
4. **배포** 클릭
5. 권한 요청 팝업 → **액세스 승인** → 계정 선택 → 허용
6. 배포 완료 후 표시되는 **URL 복사**

   ```
   예시: https://script.google.com/macros/s/AKfycbxxxxxxxxxxxxxxxx/exec
   ```

---

## 4. 관리자 페이지에 URL 등록

1. `admin.html` 접속 (비밀번호: `cynegod`)
2. 왼쪽 메뉴 → **설정** 클릭
3. **Google Sheets 연동** 카드 → Apps Script URL 입력
4. **저장** 버튼 클릭
5. **테스트 전송** 버튼으로 동작 확인
6. Google Sheets에서 테스트 행 확인 ✓

---

## 5. 수정 배포 시 주의사항

Apps Script 코드를 수정한 경우 **새 배포**를 생성해야 합니다.
기존 배포를 수정하거나 **버전 관리**를 사용하지 않으면 변경사항이 반영되지 않습니다.

1. **배포** → **배포 관리**
2. 연필 아이콘 클릭 → 버전: **새 버전**
3. **배포** → 새 URL 복사 후 관리자 페이지에 재등록

---

## 주의사항

- Apps Script 무료 할당량: **하루 최대 20,000건**
- CORS 이슈: `no-cors` 모드로 전송하므로 응답 확인 불가 → Sheets 직접 확인
- 스프레드시트 공유 설정과 무관하게 동작 (스크립트 권한으로 실행)
- 브라우저 데이터 삭제 시 관리자 설정(URL)도 초기화됨 → URL 별도 보관 권장
